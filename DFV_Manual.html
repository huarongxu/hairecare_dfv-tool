1. Overview

DFV Tool 是一个 一键运行 的自动化工具，用于执行每周 Demand Flow Verification 流程。它完全替代手动操作 SAP Analysis for Office (AO)，自动完成以下全流程：

打开 AO 工作簿
→
等待 AO 加载
→
点击 Refresh All
→
填写 Prompts
→
等待 BW 数据
→
导出 CSV
→
分析&报告
⏱ 执行时间

约 75–120 秒，取决于 SAP 数据量和网络速度。手动操作通常需要 15–30 分钟。

📊 输出内容

Excel 报告（KPI 汇总 + 按日期/按 Owner 分类的错误明细）+ CSV 原始数据 + HTML Dashboard。

🎯 核心指标

18M Volume Difference（目标 < 2%）和 13W SKU Difference（目标 = 0）。

2. Prerequisites（前提条件）

💻 系统要求
  • Windows 10/11（已加入公司域）
  • SAP Analysis for Office（AO）已安装
  • SAP SSO（单点登录）已配置
  • 屏幕分辨率 ≥ 1920×1080
🐍 Python 环境
  • Python 3.10+（推荐 3.13）
  • 安装路径不含中文/空格
  • 管理员权限安装

安装 Python 依赖

pip install -r requirements.txt

依赖包清单：

Package用途
pywinautoUI 自动化：操作 AO 的 Ribbon 按钮和 Prompts 对话框
pywin32COM 接口：连接 Excel、读取单元格数据
pandas数据处理：筛选、分类、聚合

⚠ 重要提示
运行时 不要移动鼠标或切换窗口。工具通过物理鼠标点击操作 SAP Prompts 对话框，任何鼠标干预都会导致填写错误。

Calendar Week 计算逻辑

自动计算：当前 ISO 周 到 当前 + 18 个月。例如：

  • 今天 2026-05-15 → Week 20/2026
  • +18 个月 = 2027-11 → Week 44/2027
  • Prompts 填入: 20/2026 - 44/2027

AO 工作簿路径

WORKBOOK_PATH = "...DRP Forecast Flow Validation AO (20251021) - KHP.xlsm"

如果工作簿位置或文件名变了，修改此路径即可。

💡 切换到其他 Category
只需修改 config.py 中的 CATEGORY（如 "SKINCARE"），然后重新运行即可。DRP_LOCATIONS 也需要相应调整。

5. Architecture（技术架构）

7 步执行流程

Step函数技术手段说明
1step1_open_workbook()os.startfile + COM GetObject启动 Excel，打开 AO 工作簿
2step2_wait_ao()pywinauto UIA backend等待 AO 加载完成（Analysis 标签出现）
3step3_refresh()pywinauto click_input点击 Analysis → Refresh All 按钮
4step4_fill_prompts()坐标定位 + 剪贴板粘贴填写 Region / Category / Calendar Week
5step5_wait_data()COM Range polling轮询等待 BW 数据加载完成
6step6_export()COM Range().Value 批量读取导出数据到 CSV（比逐行读快 10x）
7step7_pipeline()pandas筛选、分类错误、生成报告

核心代码文件

文件行数职责
run.py~460主入口。包含 7 个 step 函数和 UI 自动化逻辑
config.py~120配置中心。筛选条件、路径、列映射、错误类型定义
pipeline.py~320数据管线。加载 CSV → 筛选 → 分类 → KPI → 导出 Excel → 存历史 → 生成 Dashboard
history.py~160SQLite 历史存储。每次运行的 KPI + 错误明细持久化
dashboard.py~300Dashboard 生成器。输出单个 HTML 文件，含图表和交互功能

关键技术决策

🚨 为什么不用 SAP XLL 函数？

从 Python COM 调用 xl.Run("SAPExecuteCommand") 等 XLL 函数会永久破坏 AO 的 Analysis Ribbon（标签消失，需重启 Excel）。所有 SAP 交互必须通过 pywinauto 的物理按钮点击。

💡 Prompts 对话框的坐标定位

SAP AO 的 Prompts 对话框基于 WPF 技术，右侧输入面板 (DP_4) 不暴露标准 UIA 控件。工具通过固定坐标公式定位输入框：

input_y = right_panel.top + 62 + variable_index * 40

其中 variable_index 从左侧面板标签的 Y 位置自动排序得出。

💡 HKTW (5740) 说明

Location 5740 = 香港/台湾分销中心。此区域的错误通常不需要处理，在报告中标记为 "No action needed"，不计入 actionable errors。

8. Dashboard（趋势看板）

每次运行结束后，工具会自动生成 output/DFV_Dashboard.html。双击用浏览器打开即可查看。

Dashboard 功能

📊 KPI 状态卡片

Volume Diff%、Error SKUs、Demand Hub 总量、APO-DRP 总量。颜色标识 on-track / off-track。

📈 趋势图表

Volume Diff% 折线图（含 2% 目标线）+ Error SKU 柱状图（带数字标签）。位于 KPI 卡片下方、Action Items 上方。

🔎 Action Items 表格

按 Owner 过滤的错误明细表。含 Description 列。可按 Owner 快速切换筛选。

📋 Copy Table（邮件格式）

点击按钮复制表格到剪贴板，粘贴到 Outlook 邮件后保留表格格式（带表头颜色、边框）。同时支持纯文本粘贴。

📅 历史周选择器

下拉菜单切换历史周次数据，查看任意一周的 KPI 和错误详情。

🔄 Sync Owners

复制同步命令到剪贴板，在终端粘贴执行即可把 Excel 修改的 Owner 同步回 DB 并刷新 Dashboard。

数据存储

历史数据存储在 dfv_history.db（SQLite），每次运行自动追加。

Table内容
runs每次运行的 KPI 汇总（日期、周次、diff%、错误数、状态）
errors每次运行的全部错误明细（产品、位置、原因、动作、Owner）

技术实现

  • 单 HTML 文件，数据以 JSON 内嵌到 <script> 中
  • 使用 str.replace() 注入数据（避免 f-string 与 JS 大括号的冲突）
  • JS 用 ES5 语法确保兼容性（var、function 代替 const/let/arrow）
  • Chart.js 4.4.0 通过 CDN 加载（无网络时显示提示）
💡 单独重新生成 Dashboard
cd dfv_tool
python -c "from dashboard import generate_dashboard; generate_dashboard()"

10. Troubleshooting（常见问题）

❌ "Analysis ribbon did not appear within 180s"

原因：SAP AO 插件没有加载，或 SSO 登录过期。

解决：手动打开 Excel → 检查 Analysis 标签是否存在 → 确认 SAP SSO 可以正常登录 → 关闭 Excel 后重新运行。

❌ "Prompts dialog did not appear within 120s"

原因：上次的 SAP session 还有效，Refresh 直接加载数据而不弹出 Prompts。

解决：关闭 Excel → 重新运行。如果仍不弹出，手动在 AO 中点 Analysis → Prompts 验证。

❌ "Variables not accepted (OK still disabled)"

原因：Prompts 对话框的坐标偏移（屏幕分辨率/缩放设置变化）。

解决：确保显示器设置为 100% 缩放（Settings → Display → Scale = 100%）。如果使用外接显示器，确保 AO 窗口在主显示器上。

❌ 运行时鼠标被移动了

原因：Step 4 使用物理鼠标点击填写 Prompts，移动鼠标会导致点击位置偏移。

解决：运行期间（尤其是前 60 秒）不要碰鼠标和键盘。可以锁屏但不要切换用户。

❌ COM 连接失败 / "Cannot re-acquire Excel COM object"

原因：Excel COM 进程挂起或有多个 Excel 实例。

解决：打开任务管理器 → 结束所有 EXCEL.EXE 进程 → 重新运行。

⚠ 想手动单独运行 Pipeline（不刷新 SAP）

可以直接对已有的 CSV 文件运行分析：

python dfv_tool/pipeline.py --csv output/DFV_20260515_224318.csv
